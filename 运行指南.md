# CampusBook 本地运行指南（从零开始）

本文说明在新电脑上**需要安装的环境**、**推荐配置**，以及**按顺序执行的命令**，便于克隆或解压项目后能完整跑通前后端与数据库。

---

## 一、需要先安装什么

### 1. Node.js 与包管理器

- 安装 **Node.js**（建议 **20 LTS 或更高**，与仓库中 `@types/node` 一致即可）。
- 安装后会自带 **npm**，本仓库默认用 `npm` 安装依赖。

验证：

```bash
node -v
npm -v
```

### 2. MySQL 数据库（二选一）

项目使用 **Prisma + MySQL**，必须先有可用的数据库。

**方式 A（推荐）：Docker**

- 安装 **Docker Desktop**（Windows / macOS）或 **Docker Engine + Compose**（Linux）。
- 仓库根目录已有 `docker-compose.yml`，会启动 **MySQL 8.4**，端口 **3306**（账号/库名见 compose 文件与 `.env.example`）。

**方式 B：本机安装 MySQL**

- 自行安装 MySQL（版本建议 8.0+），创建数据库（例如库名 `campusbook`）及有权限的用户。
- 在 `.env` 里把 `DATABASE_URL` 改成 `mysql://用户:密码@主机:端口/库名`（可参考 `.env.example`）。

> 若本机 3306 已被占用，需改 `docker-compose.yml` 的端口映射，或改用其他 MySQL 实例，并同步修改 `DATABASE_URL`。

### 3. Git（可选）

- 若通过 `git clone` 获取代码，需要安装 **Git**；若使用压缩包解压，可跳过。

---

## 二、获取项目代码

```bash
git clone <仓库地址>
cd CampusBook
```

或解压他人发来的压缩包后，在终端进入项目根目录（包含 `package.json`、`prisma` 文件夹的那一层）。

---

## 三、环境变量

1. 复制示例配置：

   ```bash
   cp .env.example .env
   ```

2. 编辑 `.env`，至少确认：

   | 变量 | 说明 |
   |------|------|
   | `DATABASE_URL` | 与 MySQL 实际账号、密码、主机、库名一致；用本仓库自带 Docker 时通常与 `.env.example` 默认一致。 |
   | `AUTH_SECRET` | 生产级随机串；本地可用 `openssl rand -base64 32` 生成后填入。 |
   | `AUTH_URL` | 本地开发一般为 `http://localhost:3000`。 |

图片存储默认 **本地**（`public/uploads`），一般无需改 `.env`；若接 S3 / Vercel Blob，见 `.env.example` 注释与根目录 `README.md`。

---

## 四、启动数据库（若使用 Docker）

在项目根目录执行：

```bash
docker compose up -d
```

确认容器正常后，再执行后面的 `npm` / Prisma 命令。

---

## 五、安装依赖并准备数据库

**请严格按顺序执行**（先装依赖，`postinstall` 会执行 `prisma generate`）。

```bash
npm install
```

应用数据库迁移（使用仓库里已有的迁移文件）：

```bash
npx prisma migrate deploy
```

写入种子数据（测试账号与示例数据）：

```bash
npm run db:seed
```

种子数据中的默认管理员（以 `README.md` / 种子脚本为准，一般为）：

- 邮箱：`admin@campusbook.local`
- 密码：`admin123`

---

## 六、启动开发服务器

```bash
npm run dev
```

浏览器打开：**http://localhost:3000**

---

## 七、常用命令速查

| 命令 | 说明 |
|------|------|
| `npm run dev` | 本地开发（热更新） |
| `npm run build` | 生产构建（含 `prisma generate`） |
| `npm run start` | 生产模式启动（需先 `build`） |
| `npm run lint` | ESLint |
| `npm run db:migrate` | 开发时改 schema 用（`prisma migrate dev`） |
| `npm run db:push` | 仅开发快速对齐 schema，不生成迁移文件 |
| `npm run db:seed` | 重新执行种子 |
| `npm run db:studio` | 打开 Prisma Studio 浏览数据 |

## 八、常见问题

1. **`migrate deploy` 报错连不上库**  
   检查 MySQL 是否已启动、`DATABASE_URL` 是否与 Docker/本机配置一致、防火墙或端口是否冲突。

2. **图片上传失败（本地存储）**  
   确保项目根目录下 `public/uploads` 存在且进程有写权限（默认 `STORAGE_DRIVER` 为本地时）。

3. **端口 3000 被占用**  
   可用 `npx next dev -p 3001` 等方式换端口，并相应修改 `AUTH_URL` 与访问地址。

4. **仅第一次从仓库拉代码**  
   用 `npx prisma migrate deploy` 即可；若你**自己修改**了 `prisma/schema.prisma`，在开发环境应使用 `npm run db:migrate` 生成并应用新迁移。
